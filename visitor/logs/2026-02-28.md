## Unanswered Questions

1. **Who determines "quality"?** ATP is earned through "community validation" of quality contributions, but who validates? How? Is it voting? Algorithmic? Both?
2. **How does bootstrapping work?** New users start with 100 ATP and 0.5 trust. But in the Karma Journey, starting trust was 0.361. Which is it? And how do you build trust from zero with no history?
3. **What's the relationship between all these scoring systems?** T3 measures the person, V3 measures their output, CI measures consistency, ATP measures energy. How do they all interact? A visual diagram connecting these would help enormously.
4. **Is this blockchain-based?** The site carefully avoids saying "blockchain" but also doesn't say what the underlying technology IS. Is it a new protocol? A layer on existing infrastructure?
5. **Can I try this for real?** Every page says "research, not product." When might it become something I can actually use? Even a rough roadmap would help.
6. **What about privacy?** Hardware-bound identity + permanent reputation sounds like a surveillance dream. The site mentions zero-knowledge proofs but doesn't explain how privacy is preserved in practice.
7. **How does MRH work?** This acronym appears on multiple pages but I never found a clear explanation. It's the least explained of the four core mechanisms.
8. **What prevents a wealthy attacker?** The "What Could Go Wrong" page says the ROI margin between honest and dishonest strategies is small (0.93 vs 0.90). Couldn't a state actor or corporation accept that margin?

## Honest Assessment

This is an ambitious, intellectually honest project. The problem diagnosis is excellent — I viscerally felt "yes, the internet IS broken in exactly these ways." The "What Could Go Wrong" page was the best page on the site; a project that publishes its own failure analysis earns credibility.

**What works well:**
- Problem framing is superb. I understand WHY before being asked to care about WHAT.
- "A Day in Web4" is the emotional MVP. It made abstract concepts feel like improvements I actually want.
- The interactive elements (Karma Journey, Society Simulator, Playground) are engaging hooks.
- Honesty about limitations. "This is research, not a product" is refreshing in a space full of hype.
- Progressive disclosure is mostly good — the recommended path works.

**What needs work:**
- **Acronym overload is the #1 barrier.** Nine acronyms (ATP, ADP, LCT, T3, V3, CI, MRH, SAL, DID) is too many for a first visit. The site introduces them faster than I can absorb them. A persistent glossary or tooltip system would help enormously.
- **The recommended learning path isn't optimal.** "Why Web4" is too heavy as a second page. "TL;DR" or "A Day in Web4" would be gentler on-ramps.
- **Concept relationships are unclear.** Each concept is explained well in isolation, but I can't see how ATP, T3, CI, and LCT fit together as a system. A visual concept map showing "LCT enables identity → ATP tracks energy → T3 accumulates trust → CI verifies consistency" would connect the dots.
- **The "how" remains fuzzy.** I understand the philosophy and the game theory. But I don't understand what technology makes this work. Is it a protocol? A network? Software that runs on my device? The implementation layer is intentionally vague, which is fine for research, but leaves a curious visitor wanting more.

**Bottom line:** I'd return to play more with the simulators. I'd recommend this to someone interested in internet governance or trust systems, but I'd warn them: "Be ready for a lot of acronyms. Start with the 'Day in Web4' page, not the homepage's suggested path."
